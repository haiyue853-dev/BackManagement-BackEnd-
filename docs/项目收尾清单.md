# 项目收尾清单

这份清单用于项目最后验收，确保当前全栈后台项目已经达到“可演示、可交付、可继续维护”的状态。

## 1. 后端检查

- [ ] `.env` 已按 `.env.example` 配置完成
- [ ] MySQL 连接正常
- [ ] `npm test` 全部通过
- [ ] `GET /health` 返回正常
- [ ] 登录接口正常
- [ ] 用户、商品、账号、个人中心接口都能访问

## 2. 前端检查

- [ ] `npm run build` 通过
- [ ] 登录页正常
- [ ] 首页数据正常显示
- [ ] 用户管理页面增删改查正常
- [ ] 商品管理页面增删改查正常
- [ ] 个人中心保存正常
- [ ] 401 失效时会自动回到登录页

## 3. 联调检查

- [ ] 管理员账号能访问账号管理
- [ ] 普通编辑账号不能访问账号管理
- [ ] 商品状态使用 `上架 / 下架`
- [ ] 搜索统一使用 `keyword`
- [ ] 分页参数 `page / pageSize` 正常

## 4. 文档检查

- [ ] 根目录 README 已说明项目结构
- [ ] 后端 README 已说明运行方式
- [ ] 前端 README 已说明联调方式
- [ ] 后端接口文档已更新
- [ ] 后端部署说明已补齐

## 5. Git 检查

- [ ] 前端仓库工作区干净
- [ ] 后端仓库工作区干净
- [ ] 关键节点都已 commit
- [ ] 远端 push 成功

## 6. 下一阶段建议

如果以上都完成，项目已经可以进入下一阶段：

1. 演示与答辩
2. 简历项目整理
3. 云服务器部署
4. 生产化增强
