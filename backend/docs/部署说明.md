# 后端部署说明

本文档用于说明后端服务在本地、测试环境或云服务器上的基础部署方式。

## 1. 运行环境

- Node.js 18 及以上
- MySQL 8.x
- npm
- Windows 或 Linux 均可

## 2. 环境变量

启动前请先在 `backend` 目录下创建 `.env` 文件，可参考 `.env.example`：

```env
PORT=3000
JWT_SECRET=replace-with-a-secure-secret
JWT_EXPIRES_IN=7d
DB_SYNC_ENABLED=true
DB_SYNC_ALTER=false
DB_HOST=localhost
DB_PORT=3306
DB_NAME=dev5
DB_USER=root
DB_PASSWORD=123456
LOG_LEVEL=info
```

生产环境建议：

- 使用更复杂的 `JWT_SECRET`
- 不要把真实密码提交到仓库
- `DB_SYNC_ALTER=false`
- `DB_SYNC_ENABLED` 仅在确认需要自动建表时开启

## 3. 本地部署

安装依赖：

```bash
npm install
```

开发启动：

```bash
npm run dev
```

正式启动：

```bash
npm start
```

## 4. PM2 部署

如果在服务器上使用 PM2，可以执行：

```bash
npm install
pm2 start bin/www --name backend-management-api
```

查看状态：

```bash
pm2 list
```

查看日志：

```bash
pm2 logs backend-management-api
```

重启服务：

```bash
pm2 restart backend-management-api
```

## 5. 健康检查

部署后建议先访问：

```http
GET /health
```

如果返回 `status: ok` 且 `database: connected`，说明服务和数据库连接正常。

## 6. 反向代理建议

如果前端和后端分开部署，可以使用 Nginx 做反向代理：

- 前端静态资源走前端站点
- `/api` 转发到 `http://127.0.0.1:3000`

这样前端仍然可以保持：

```env
VITE_API_BASE=/api
```

## 7. 部署后检查清单

- 能访问 `/health`
- 能成功登录
- 首页数据能正常加载
- 用户管理接口能正常查询
- 商品管理接口能正常查询
- 账号管理只有管理员能访问
- 个人中心能正常保存资料
- 服务日志中没有连续报错
